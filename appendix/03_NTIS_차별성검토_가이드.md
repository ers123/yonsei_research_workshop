# 별첨 3: NTIS 활용 — AI 에이전트로 차별성검토 효율화하기
**연구과제 신청 시 국가 R&D 중복성/차별성 검토 실무 가이드**

> NTIS(국가과학기술지식정보서비스, ntis.go.kr)는 한국 정부가 운영하는 국가 R&D 과제 통합 데이터베이스입니다.

---

## NTIS가 필요한 이유

연구과제 신청서를 작성할 때 **차별성검토**(중복성 검토)는 필수입니다. 심사위원은 "기존 국가 R&D 과제와 뭐가 다른가?"를 반드시 확인합니다. 이때 근거 자료의 출처가 NTIS입니다.

구체적으로:
- **NRF(한국연구재단)** 과제 신청 시 차별성검토 기술서 작성 필수
- **IITP, KETEP, KAIA** 등 부처 R&D도 유사과제 검토를 요구
- 심사위원이 직접 NTIS에서 교차 확인하므로, 검토 누락은 감점 사유

---

## 방법 1: NTIS 웹 직접 검색 (기본)

### Step 1: NTIS 접속 및 과제 검색

1. **ntis.go.kr** 접속
2. 상단 메뉴에서 **"과제정보"** 클릭 (또는 검색창에서 직접 키워드 입력)
3. **"국가R&D과제"** 탭 선택

### Step 2: 검색 키워드 전략

단순 키워드 하나로는 부족합니다. 다음과 같이 조합합니다:

| 전략 | 예시 |
|---|---|
| **핵심 키워드** | "자율주행 센서 융합" |
| **유사어/동의어** | "자율주행" → "무인운전", "자동운전" |
| **영문 키워드** | "autonomous driving sensor fusion" |
| **세부 기술 키워드** | "LiDAR 포인트클라우드", "카메라-라이다 정합" |
| **응용 분야** | "스마트 모빌리티", "교통 안전" |

### Step 3: 검색 결과 필터링

- **수행 기간**: 최근 5년 이내 (종료 과제 포함)
- **과제 상태**: 수행중 + 완료
- **연구비 규모**: 본인 신청 규모와 유사한 범위
- **부처**: 관련 부처 선택 (과기정통부, 산업부 등)

### Step 4: CSV 다운로드

검색 결과 화면 하단의 **"엑셀 다운로드"** 버튼으로 결과를 CSV/Excel 파일로 내려받습니다. 이 파일이 AI 에이전트 분석의 입력 자료가 됩니다.

---

## 방법 2: AI 에이전트로 검색 결과 분석하기

NTIS는 공개 API를 제공하지 않으므로, 웹에서 직접 검색하고 결과를 다운로드하는 것이 현실적인 접근법입니다. AI 에이전트의 역할은 **다운로드한 데이터의 분석과 차별성 기술서 초안 작성**입니다.

### 작업 흐름

```
[연구자] NTIS 웹 검색 → CSV 다운로드
    ↓
[AI 에이전트] CSV 분석 → 유사과제 분류 → 차별성 매트릭스 작성
    ↓
[연구자] 검토 및 수정 → 신청서 반영
```

### 프롬프트 예시: CSV 분석 요청

```
첨부한 NTIS_검색결과.csv 파일을 분석해주세요.

내 연구 주제: [연구 주제 한 문장]
내 연구의 핵심 차별점: [차별점 2-3개]

다음을 수행해주세요:
1. 유사도가 높은 과제 상위 10개를 선별하고, 각각 한 줄 요약
2. 각 유사과제와 내 연구의 차이점을 표로 정리
3. 차별성검토 기술서 초안을 작성 (NRF 양식 기준)
```

### 프롬프트 예시: 차별성 매트릭스 생성

```
아래 NTIS 유사과제 목록을 바탕으로 차별성 매트릭스를 만들어주세요.

비교 축:
- 연구 목적
- 방법론/접근법
- 적용 대상/범위
- 기대 성과
- 연구비 규모 및 기간

내 과제 정보:
- 과제명: [과제명]
- 목적: [목적]
- 방법론: [방법론]
- 기간/규모: [기간/규모]

출력 형식: 마크다운 표
```

### Claude Code 활용 (로컬 파일 분석)

Claude Code 환경에서는 CSV 파일을 프로젝트 폴더에 넣고 바로 분석할 수 있습니다:

```bash
# 프로젝트 폴더에 NTIS 다운로드 파일 복사
cp ~/Downloads/NTIS_검색결과.xlsx ./data/

# Claude Code에서 분석 요청

#  유사한 과제를 분류해줘"
```

## 실무 팁

### 검색은 넓게, 분석은 좁게

- NTIS 검색 시에는 키워드를 **넓게** 잡아서 누락을 방지합니다 (검색 결과 100건 이상 확보 권장)
- AI 에이전트에게는 **좁은 기준**으로 유사도 상위 과제만 선별하도록 지시합니다

### 검색 이력을 남겨라

- 어떤 키워드로, 언제 검색했는지 기록해두면 심사 시 "충분히 검토했다"는 근거가 됩니다
- 검색 키워드 목록 + 검색일자 + 결과 건수를 차별성검토서에 명시하면 신뢰도가 올라갑니다

### NTIS 외 보완 검색

| 출처 | 용도 | URL |
|---|---|---|
| **NTIS** | 국가 R&D 과제 (핵심) | ntis.go.kr |
| **RISS** | 국내 학위논문/학술지 | riss.kr |
| **KCI** | 한국학술지인용색인 | kci.go.kr |
| **NDSL/ScienceON** | 국내외 과학기술 정보 | scienceon.kisti.re.kr |

## 주의사항

- **NTIS에는 공개 API가 없습니다.** 웹 검색 + 수동 다운로드가 유일한 실용적 방법입니다. 간혹 비공식 스크래핑을 시도하는 경우가 있으나, 이용약관 위반 가능성이 있으므로 권장하지 않습니다.
- **CSV 데이터의 한계**: 다운로드 가능한 필드(과제명, 연구책임자, 연구비, 기간 등)는 요약 정보입니다. 상세 연구 내용은 NTIS에서 개별 과제 페이지를 확인해야 합니다.
- **최신성 확인**: NTIS 데이터는 등록 시점 기준이므로, 가장 최근 공고된 과제는 아직 반영되지 않았을 수 있습니다.
