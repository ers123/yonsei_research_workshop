# 별첨 1: 내 노트북에서 무료로 AI 돌리기
**문과·사회과학 대학원생을 위한 로컬 AI 설치 가이드 (Ollama 기준)**

> 대상: 터미널·Node.js·Python 모두 처음인 석·박사생.
> 목표: 이 문서를 따라 10~15분 안에 내 PC에서 무료로 돌아가는 AI 한 대를 확보한다.

---

## 들어가며: 왜 로컬 AI인가

Claude나 ChatGPT 같은 "프런티어(frontier) 모델"은 똑똑하지만 두 가지 제약이 있습니다.

1. **돈** — 유료 구독 또는 API 비용이 듭니다.
2. **프라이버시** — IRB 승인 인터뷰 녹취록, 미공개 설문 원자료, 환자/학생 데이터를 외부 서버로 보내는 것은 대부분의 연구윤리 규정에서 금지됩니다.

**로컬 AI는 내 노트북 안에서만 돌아가는 AI 모델입니다.** 인터넷 없이 작동하고, 데이터가 PC를 떠나지 않으며, 비용은 전기세뿐입니다. 대신 프런티어 모델보다 덜 똑똑하고, 모델 크기에 따라 노트북이 뜨거워집니다.

이 가이드에서는 가장 보편적인 도구인 **Ollama** 하나만 설치합니다. Ollama는 "로컬에서 AI 모델을 실행하는 가장 간단한 도구"이며, 백그라운드에서 조용히 돌아갑니다.

> 💡 이 별첨은 "설치"까지만 다룹니다. 설치 후 Claude Code와 어떻게 엮을지, RA(연구 조수) 팀을 어떻게 구성할지는 `hands_on/prompts/track_c_연구조수팀.md`와 `hands_on/ra_team_setup.md`에서 이어집니다.

## Step 1 — 내 노트북이 감당할 수 있는지 먼저 확인

AI 모델은 RAM(메모리)을 많이 먹습니다. **설치 전에 본인 노트북 사양을 확인**하세요.

### 1-1. 메모리 확인

```bash
# macOS (터미널 앱에서)
system_profiler SPHardwareDataType | grep "Memory"
```

```powershell
# Windows (PowerShell에서)
Get-CimInstance Win32_ComputerSystem | Select-Object TotalPhysicalMemory
```

결과에서 "Memory" 또는 숫자(바이트)를 확인합니다. Windows 결과는 아주 큰 숫자로 나오는데, `/1024/1024/1024` 를 해보면 GB 단위가 됩니다. 대략:
- 약 `17179869184` → 16GB
- 약 `25769803776` → 24GB
- 약 `34359738368` → 32GB

### 1-2. 내 환경에 맞는 모델 미리보기

| 내 노트북 | 현실적 선택 | 기대치 |
|---|---|---|
| **RAM 8GB** | 아쉽게도 안정적으로 돌리기 어려움 | 프런티어 무료 플랜(Claude, Gemini)을 쓰세요 |
| **RAM 16GB (맥북 에어 / 일반 윈도우)** | **Gemma 3 4B** 또는 Phi-4-mini | 초안 요약, 간단한 질의응답, 분류 |
| **RAM 24~32GB** | Qwen 2.5 14B (Q4) | 한국어 연구 글쓰기 보조, 코드북 재구성 |
| **RAM 32GB+ / 데스크톱** | Qwen3-Coder 30B, Gemma 3 27B | 본격적인 연구 조수 활용 |

⚠️ **중요**: "크면 클수록 좋다"가 아닙니다. 16GB에 14B 모델을 무리해서 돌리면 노트북이 먹통이 되고, 다른 앱(브라우저·Zotero·한글)과 동시에 못 씁니다. **일단 작은 것부터 시작하세요.**

### ✔ 체크포인트

- [ ] 내 노트북 RAM이 몇 GB인지 숫자로 알고 있다
- [ ] 위 표에서 내가 시도할 "1순위 모델"을 하나 고를 수 있다

## Step 2 — Ollama 설치

### 2-1. macOS

1. 브라우저에서 **https://ollama.com/download** 접속
2. "Download for macOS" 클릭 → `Ollama-darwin.zip` 다운로드
3. 압축 풀고 `Ollama.app` 을 `/Applications` 폴더로 드래그
4. `Ollama.app` 실행 → 메뉴바 오른쪽 위에 라마 아이콘이 뜨면 성공

설치 확인:
```bash
# macOS
ollama --version
```

`ollama version is 0.x.x` 비슷한 문구가 뜨면 성공입니다.

### 2-2. Windows

1. 브라우저에서 **https://ollama.com/download** 접속
2. "Download for Windows" 클릭 → `OllamaSetup.exe` 다운로드
3. 설치 마법사 실행 → 기본값으로 "Next" 계속 클릭
4. 설치 끝나면 시스템 트레이(시계 옆)에 라마 아이콘이 뜸

설치 확인:
```powershell
# Windows (PowerShell)
ollama --version
```

⚠️ **"ollama 라는 명령을 찾을 수 없습니다" 에러가 나면** 설치 후 PowerShell 창을 **완전히 닫고 새로 열어주세요**. PATH 환경변수가 새 창에만 반영됩니다. 그래도 안 되면 재부팅하면 거의 해결됩니다.

⚠️ **Windows 보안 경고**: PowerShell에서 처음 명령 실행 시 "이 시스템에서 스크립트를 실행할 수 없습니다" 같은 경고가 나올 수 있습니다. PowerShell을 **관리자 권한**으로 열어 아래 한 번 실행한 뒤 다시 시도하세요:
```powershell
# Windows (PowerShell, 관리자 권한)
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### ✔ 체크포인트

- [ ] 터미널(또는 PowerShell)에서 `ollama --version` 을 쳤을 때 버전 번호가 나온다
- [ ] 메뉴바(macOS) 또는 시스템 트레이(Windows)에 라마 아이콘이 보인다

## Step 3 — 모델 다운로드 & 첫 대화

Ollama는 설치만으로는 아무것도 못 합니다. **AI 모델(파일)을 따로 받아야** 작동합니다.

### 3-1. 권장: 가장 작고 빠른 것부터

RAM 16GB 이하라면 Gemma 3 4B 또는 Phi-4-mini로 시작하세요. 4~5GB를 받습니다.

```bash
# macOS / Windows 공통 (터미널 또는 PowerShell)
ollama pull gemma3:4b
```

"pull" 은 "내려받다"라는 뜻입니다. 다운로드에 몇 분 걸립니다 (인터넷 속도 의존). 진행률이 표시됩니다.

### 3-2. 첫 대화 해보기

```bash
# macOS / Windows 공통
ollama run gemma3:4b
```

프롬프트(`>>>`)가 뜨면 한국어로 말을 걸어보세요:

```
>>> 안녕하세요. 스스로를 한 문장으로 소개해주세요.
```

AI가 답을 합니다. 대화 종료는 `/bye` 입력 또는 `Ctrl+D`.

⚠️ **"model requires more system memory" 에러가 나면**: 모델이 노트북에게 너무 큽니다. `ollama pull phi4-mini` 로 더 작은 모델을 받아 다시 시도하세요.

⚠️ **엄청 느리거나 노트북 팬이 최대로 돌면**: 정상입니다. 다만 브라우저·Zoom 같은 무거운 앱을 먼저 끄면 훨씬 빨라집니다.

### 3-3. (선택) 다른 모델 추가로 받아보기

나중에 비교해보고 싶으면:

```bash
# macOS / Windows 공통
ollama pull phi4-mini           # MS Phi-4 mini (3.8B, 함수 호출 잘함)
ollama pull qwen2.5:14b         # Qwen 2.5 14B (한국어 품질 ↑, RAM 24GB+ 권장)
```

설치한 모델 목록 보기:

```bash
ollama list
```

모델 삭제 (용량 정리):
```bash
ollama rm gemma3:4b
```

### ✔ 체크포인트

- [ ] `ollama list` 로 다운로드된 모델이 최소 1개 보인다
- [ ] `ollama run <모델명>` 으로 한국어 대화가 된다
- [ ] `/bye` 로 프롬프트를 빠져나올 수 있다

## Step 4 — 연구에 써보기 (바로 쓰는 3가지 예시)

여기까지 왔다면 이미 "내 PC에서 AI 한 대가 돌아가는" 상태입니다. 아래는 설치 직후 바로 시험해볼 수 있는 연구 활용 예시입니다.

### 4-1. IRB 인터뷰 녹취록 1차 코딩

```
ollama run gemma3:4b
```
```
>>> 다음은 대학원생과의 반구조화 인터뷰 발췌입니다.
>>> 근거이론(Grounded Theory) 접근으로 개방 코딩(open coding)을 해주세요.
>>> 코드명 + 해당 발화 인용 형식으로 정리해주세요.
>>>
>>> [여기에 녹취록 붙여넣기]
```

**왜 로컬이 필요한가**: IRB 승인 데이터는 클라우드 전송 자체가 윤리 위반입니다. 이게 로컬 AI의 진짜 가치입니다.

### 4-2. 논문 PDF 요약 (텍스트 복사 후)

PDF 본문을 복사해 붙여넣으면 됩니다:
```
>>> 다음 논문 본문을 (1) 연구 질문 (2) 방법 (3) 핵심 발견 (4) 한계 의 4개 항목으로 요약해주세요.
>>> [본문 붙여넣기]
```

### 4-3. 한국어 초안 다듬기

```
>>> 다음 문단을 학술적 한국어로 자연스럽게 다듬어주세요. 의미는 유지.
>>> [문단 붙여넣기]
```

## "로컬로 할까, Claude Pro/ChatGPT 로 할까?" — 판단 기준

완벽한 답은 없습니다. 상황별로 나눠 쓰는 게 현실적입니다.

| 작업 종류 | 추천 | 이유 |
|---|---|---|
| IRB 인터뷰 녹취록 분석 | **로컬(Ollama)** | 프라이버시. 외부 전송 불가. |
| 미공개 설문 원자료 요약 | **로컬(Ollama)** | 동일. |
| 공개 논문 문헌 검토 | **프런티어(Claude/GPT)** | 민감자료 아님. 품질이 압도적. |
| 연구계획서 초안 글쓰기 | **프런티어** | 한국어 글쓰기 품질 차이가 큼. |
| 대량 분류·태깅 반복 작업 | **로컬** | 속도는 느려도 $0. 밤새 돌려도 됨. |
| 코드 짜기 (통계·크롤러) | **프런티어** (가능하면) | 복잡한 코드는 여전히 프런티어 우위. |
| 오프라인 출장·학회 | **로컬** | 인터넷 없어도 됨. |

**실전 팁**: 로컬에서 1차로 민감자료를 익명화·요약한 뒤, 그 결과물만 프런티어에게 넘겨 마무리하는 "파이프라인" 이 가장 효율적입니다. 이 패턴은 Track C 에서 본격적으로 다룹니다.

## 문제 해결 (FAQ)

**Q1. 다운로드 중간에 멈춰요.**
- Ctrl+C 로 중단하고 `ollama pull <모델명>` 을 다시 실행하면 이어받기 됩니다.

**Q2. 대답이 영어로만 나와요.**
- 프롬프트 맨 앞에 "한국어로 답변하세요." 를 명시하세요. 작은 모델일수록 이 지시를 잊어버립니다.

**Q3. "context length exceeded" 에러가 떠요.**
- 입력이 모델의 컨텍스트 한도를 넘었습니다. 녹취록을 여러 개로 나누어 넣거나, 더 큰 컨텍스트를 가진 모델(`qwen2.5:14b` 등)로 바꾸세요.

**Q4. Ollama 가 백그라운드에서 너무 메모리를 먹어요.**
- 메뉴바(macOS)/트레이(Windows) 의 라마 아이콘 → Quit 으로 종료하면 메모리가 반환됩니다.

**Q5. 저장 공간이 부족해요.**
- `ollama list` 로 확인하고 `ollama rm <모델명>` 으로 안 쓰는 모델을 지우세요. 모델 하나당 2~20GB 를 차지합니다.

**Q6. GPU 가 있는데 활용되고 있나요?**
- macOS (Apple Silicon) 는 자동으로 GPU(Metal) 를 씁니다. Windows + NVIDIA GPU 도 자동으로 CUDA 를 씁니다. 별도 설정 불필요. `ollama run` 중에 작업 관리자를 열어보면 GPU 사용률이 오르는 것을 확인할 수 있습니다.
