# 별첨 1: Claude Code를 무료로 쓰는 두 가지 방법
**대학원생을 위한 로컬 AI & 무료 클라우드 활용 가이드 (Windows 환경 기준)**

> 원문: "Running Claude Code for Free: Two Methods" by Nate Herk (AI Automation Society)
> 번역 + 연구 활용 보충: 이요한 | HarmonyOn

---

## 들어가며

Claude Code의 터미널 환경, 프로젝트 관리, CLAUDE.md 파일, MCP 연동 등 모든 기능은 Anthropic 호환 API 엔드포인트라면 어디든 연결됩니다. 즉, 유료 Anthropic 백엔드를 로컬 오픈소스 모델이나 무료 클라우드 라우터로 대체할 수 있습니다. 둘 다 비용은 0원입니다.

---

## 방법 1: Ollama (로컬 모델)

**적합한 경우**: 프라이버시 중시, 오프라인 사용, 사용량 제한 없음, 비용 0원
**트레이드오프**: 하드웨어 성능에 의존. 더 크고 똑똑한 모델은 더 많은 RAM/VRAM 필요. 단, Ollama에는 클라우드 모델도 일부 제공되므로, 그쪽을 쓰면 완전한 로컬/프라이버시는 아니게 됩니다.

### Step 1: Ollama 설치

ollama.com에서 본인 OS(macOS, Windows, Linux)에 맞는 버전을 다운로드합니다. 설치 후 Ollama는 백그라운드에서 자동 실행됩니다.

### Step 2: 하드웨어에 맞는 모델 선택 — **"얼마나 큰 모델인가"가 아니라 "내 노트북에서 안정적으로 돌아가고 Claude Code가 요구하는 것을 만족하는가"**

대학원생용 모델 선택의 핵심은 세 가지입니다:

1. **64K+ 컨텍스트** — Claude Code는 최소 이 수준을 요구합니다. 작은 컨텍스트는 파일 작업 도중 맥락을 잃습니다.
2. **도구 호출(tool calling) 안정성** — agentic framework의 핵심. 작은 모델도 이게 되면 유용, 큰 모델도 안 되면 무의미.
3. **파일 작업 지속성** — 여러 턴에 걸쳐 파일을 읽고 수정하는 작업에서 일관성 유지.

즉 "단순 채팅이 되느냐"보다 **agentic 안정성**이 우선입니다.

#### 환경별 권장 구성

| 환경 | 1순위 모델 | 보조 모델 | 상위 시도 |
|---|---|---|---|
| **맥북 에어 16GB, 윈도우 16GB 저사양** | Gemma 4 E2B | Phi-4-mini-instruct | (불가) |
| **맥북 24-32GB, 윈도우 16-24GB 여유** | Gemma 4 E4B | Phi-4-mini-instruct | Qwen 2.5 14B (Q4) |
| **맥북 32GB+ / 데스크톱 / 연구실 PC** | Qwen3-Coder 30B | Devstral Small 2 | Gemma 3 27B |

#### 모델별 포지션

| 모델 | 파라미터 | 실메모리(4-bit) | 한국어 | 도구 호출 | 가장 잘 하는 일 |
|---|---|---|---|---|---|
| **Gemma 4 E2B** | 2.3B | ~3.2GB | 약함 | ⚠️ | 온디바이스, 전사 후 정리, 분류. 멀티모달 지원. |
| **Gemma 4 E4B** | 4B | ~5GB | 보통 | ✅ | 연구 보조, function calling, 멀티모달. 노트북 친화. |
| **Phi-4-mini-instruct** | 3.8B | ~4GB | 보통 | ✅ | 함수 호출 강함, 다국어. agentic 보조 모델로 적합. |
| **Qwen 2.5 14B (Q4)** | 14B | ~8GB | **강함** | ✅ | 한국어 연구 글쓰기 하한선. codebook 재구성 가능. |
| **Qwen3-Coder 30B** | 30B | ~18GB | 양호 | ✅ | "most agentic code model" 자칭. 256K 컨텍스트. |
| **Devstral Small 2 (Mistral)** | 24B | ~15GB | 양호 | ✅ | agentic coding 전용 포지셔닝, 256K 컨텍스트. |
| **Gemma 3 27B** | 27B | ~16GB | 양호 | ✅ | Opus급은 아니지만 codebook 정제까지 가능. |

**일반화가 위험한 지점:**
- 과거 가이드의 "16GB → 14B-20B"는 실제 agentic 사용(KV cache + 동시 앱)에서 버겁습니다.
- 안전한 표현: **"16GB 노트북은 E2B/E4B + Phi-4-mini 중심, 32GB부터 Qwen3-Coder/Devstral 상위권 검토."**

#### 강의용 한 문단 요약

> "대학원생의 개인 노트북에서 연구 보조용 로컬 AI를 돌리려면, 최신 고성능 모델 하나를 무리해서 쓰기보다 작은 멀티모달 모델과 긴 컨텍스트를 지원하는 모델을 선택하는 것이 현실적입니다. 대부분의 학생에게는 **Gemma 4 E2B/E4B 또는 Phi-4-mini**가 적합하고, 32GB 이상 고사양 환경에서는 **Qwen3-Coder나 Devstral Small 2** 같은 상위 agentic coding 모델을 고려할 수 있습니다."

#### Claude Code 호환성 체크리스트

새 모델을 써보기 전에 확인:
- [ ] 컨텍스트 64K 이상? (128K+ 권장)
- [ ] Tool calling 네이티브 지원? (Anthropic 호환 API로 매핑 가능?)
- [ ] Ollama/OpenRouter에서 `:instruct` 또는 `:chat` 태그 존재?
- [ ] 한국어 agentic 작업 시연 영상/벤치마크 공개?

### Step 3: 모델 다운로드

```shell
ollama pull qwen3-coder
```

로컬에 모델을 다운로드합니다. 한 번만 받으면 이후 계속 사용 가능합니다.

**클라우드 단축키**: Ollama는 클라우드 호스팅 모델도 제공합니다 (예: `glm-5:cloud`, `kimi-k2.5:cloud`). 로컬 다운로드 없이 바로 실행됩니다. 무료 티어는 제한적이지만 시작하기엔 충분합니다.

### Step 4: Claude Code를 Ollama에 연결

**옵션 A, 가장 간단 (Ollama v0.15+):**

```shell
ollama launch claude
```

Ollama가 모든 환경 변수를 자동 설정하고 Claude Code를 실행합니다.

**옵션 B, 수동 (모든 버전):**

```shell
export ANTHROPIC_BASE_URL="http://localhost:11434"
export ANTHROPIC_AUTH_TOKEN="ollama"
export ANTHROPIC_API_KEY=""

claude --model qwen3-coder
```

**옵션 C, 영구 설정 (~/.claude/settings.json):**

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "http://localhost:11434",
    "ANTHROPIC_AUTH_TOKEN": "ollama",
    "ANTHROPIC_API_KEY": ""
  }
}
```

### Step 5: 최초 로그인

Claude Code가 로그인을 요청하면 **"Anthropic Console account (API usage billing)"**을 선택합니다. 크레딧이 없어도 괜찮습니다. Ollama와 통신하기 위한 API 흐름을 활성화하는 것뿐입니다.

### Anthropic 모델 대비 잃는 것

- **네이티브 웹 검색**: MCP 기반 검색 도구(Brave, Tavily)로 대체 가능
- **프롬프트 캐싱**: 매 턴마다 전체 컨텍스트를 다시 읽음 (비용이 $0이니 괜찮음)
- **절대적 성능**: 오픈소스 모델은 훌륭하지만 Opus 4.6 티어에는 미치지 못함

### 다시 Anthropic으로 전환하려면

```shell
unset ANTHROPIC_BASE_URL
unset ANTHROPIC_AUTH_TOKEN
unset ANTHROPIC_API_KEY
claude
```

## 방법 2: OpenRouter (무료 클라우드 모델)

**적합한 경우**: 로컬보다 나은 모델 품질, 하드웨어 제약 없음, 클라우드 속도
**트레이드오프**: 사용량 제한 있음, 인터넷 연결 필요

### Step 1: OpenRouter 설정

1. openrouter.ai에서 계정 생성
2. API 키 발급
3. **$10 크레딧 추가** — 무료 모델 요청이 하루 1,000건으로 늘어남 (미추가 시 50건). $10은 유료 모델 사용 시에만 소비됩니다. 무료(`:free`) 모델은 항상 $0.

| 계정 상태 | 무료 모델 일일 요청 |
|---|---|
| 크레딧 없음 | 50/day |
| $10+ 크레딧 | 1,000/day |

### Step 2: 무료 모델 선택

OpenRouter에서 `:free` 태그가 붙은 모델을 찾으세요. 2026년 4월 기준 추천:

| 모델 ID | 컨텍스트 | 특징 |
|---|---|---|
| `qwen/qwen3-coder:free` | 262K | 종합 최고 |
| `mistralai/devstral-2:free` | 256K | 멀티파일 에이전틱 작업 |
| `moonshotai/kimi-k2.5:free` | 256K | 강력한 범용 에이전트 |
| `deepseek/deepseek-v3.1-nex-n1:free` | 131K | 에이전트 최적화 |

`openrouter/auto:free`는 피하세요. 도구 호출이 부실한 모델로 라우팅될 수 있습니다.

### Step 3: 6개 모델 슬롯 모두 설정

Claude Code는 내부적으로 6개의 모델 슬롯을 사용합니다. 설정하지 않은 슬롯은 자동으로 유료 Anthropic 모델로 라우팅됩니다. 반드시 전부 설정하세요.

**~/.zshrc 또는 ~/.bashrc에 추가:**

```shell
export ANTHROPIC_BASE_URL="https://openrouter.ai/api"
export ANTHROPIC_AUTH_TOKEN="sk-or-your-key-here"
export ANTHROPIC_API_KEY=""

export ANTHROPIC_MODEL="qwen/qwen3-coder:free"
export ANTHROPIC_DEFAULT_SONNET_MODEL="qwen/qwen3-coder:free"
export ANTHROPIC_DEFAULT_OPUS_MODEL="qwen/qwen3-coder:free"
export ANTHROPIC_DEFAULT_HAIKU_MODEL="qwen/qwen3-coder:free"    # ← 가장 흔한 과금 유출 지점
export ANTHROPIC_SMALL_FAST_MODEL="qwen/qwen3-coder:free"
export CLAUDE_CODE_SUBAGENT_MODEL="qwen/qwen3-coder:free"

export CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1
```

**Haiku 슬롯이 중요한 이유**: `ANTHROPIC_DEFAULT_HAIKU_MODEL`은 매 프롬프트마다 세션 제목 생성을 위해 백그라운드 호출됩니다. 가장 흔한 과금 유출 지점입니다.

**또는 ~/.claude/settings.json에 설정:**

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "https://openrouter.ai/api",
    "ANTHROPIC_AUTH_TOKEN": "sk-or-your-key-here",
    "ANTHROPIC_API_KEY": "",
    "ANTHROPIC_MODEL": "qwen/qwen3-coder:free",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "qwen/qwen3-coder:free",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "qwen/qwen3-coder:free",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "qwen/qwen3-coder:free",
    "ANTHROPIC_SMALL_FAST_MODEL": "qwen/qwen3-coder:free",
    "CLAUDE_CODE_SUBAGENT_MODEL": "qwen/qwen3-coder:free",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": "1",
    "CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS": "1"
  }
}
```

## 워크숍 보충: 연구에 어떻게 활용할 것인가

위 설정이 끝나면, Claude Code의 모든 기능 — 파일 읽기/쓰기, 코드 실행, MCP 도구 연결 — 을 무료로 사용할 수 있습니다. 아래는 대학원 연구에 바로 적용 가능한 시나리오입니다.

### 시나리오 1: 인터뷰 녹취록 질적 코딩 (IRB 데이터)

IRB 승인 인터뷰 데이터는 클라우드에 올릴 수 없습니다. Ollama 로컬 모델로 해결합니다.

```
# Ollama + Claude Code 로컬 설정 후
claude

> 이 폴더의 인터뷰 녹취록 20개를 읽고,
> 근거이론(Grounded Theory)으로 주제(theme)를 추출해줘.
> 각 주제별로 대표 발화를 인용하고, codebook.md로 정리해줘.
```

데이터가 내 PC를 벗어나지 않으면서 AI의 분석 능력을 활용할 수 있습니다.

### 시나리오 2: 논문 100편 배치 분류

체계적 문헌 고찰(Systematic Review)을 위한 대량 논문 분류. OpenRouter 무료 모델로 처리합니다.

```
# OpenRouter 설정 후
claude

> papers/ 폴더에 있는 논문 PDF 100개를 읽고,
> 각 논문의 (1)연구방법 (2)주요발견 (3)한계점을 추출해서
> review_matrix.csv로 정리해줘.
> PRISMA 흐름도도 mermaid로 작성해줘.
```

하루 1,000건 무료 요청이면 논문 100편 처리에 충분합니다.

### 시나리오 3: 연구계획서 초안 + NTIS 차별성 분석

```
claude

> NTIS에서 "[내 연구 키워드]"로 검색한 기존 과제 목록이 ntis_results.txt에 있어.
> 이 과제들의 공통 방향과 빈 영역(gap)을 분석하고,
> 차별화된 연구계획서 목차를 제안해줘.
> 각 장의 핵심 논점과 차별성 근거를 포함해줘.
```

### 시나리오 4: 프로젝트 맥락 문서(CLAUDE.md)로 장기 프로젝트 관리

대화가 길어지면 AI도 맥락을 잃습니다. CLAUDE.md 파일에 프로젝트 맥락을 기록해 두면, 매 세션마다 AI가 이를 읽고 이어서 작업합니다.

```markdown
# 프로젝트 맥락 (CLAUDE.md)

## 목적
이 프로젝트는 [연구 주제]에 대한 석사 논문 작성이다.

## 현재 상태
- 완성: 문헌 검토, 목차
- 작성 중: 서론
- 미완성: 방법론, 결론

## 주의사항
- NTIS 차별성 검토에서 유사 과제 3건 확인됨. 차별화 필수.
- 지도교수 피드백: 이론적 프레임워크 보강 필요
```

## 방법 1 vs 방법 2: 어떤 걸 선택할까?

| | 방법 1: Ollama (로컬) | 방법 2: OpenRouter (클라우드) |
|---|---|---|
| 비용 | 완전 무료 | 무료 (일 1,000건 제한) |
| 프라이버시 | 데이터가 PC를 안 떠남 | 데이터가 클라우드 거침 |
| 성능 | 하드웨어 의존 | 클라우드 서버 성능 |
| 인터넷 | 불필요 | 필수 |
| 모델 품질 | 좋음 (하드웨어가 충분하면) | 더 좋음 (클라우드 최적화) |
| **추천 대상** | **민감 데이터 (IRB, 인터뷰)** | **일반 연구 작업** |

**결론: 둘 다 설정해 두고, 상황에 따라 전환하는 것이 최선입니다.**
민감 데이터 → Ollama, 일반 작업 → OpenRouter.
