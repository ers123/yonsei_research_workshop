# 프로젝트 컨텍스트: 출연연 국제 R&D 협력 질적 연구

## 프로젝트 목표

한국 정부출연연구기관(출연연) 15개 기관의 국제 R&D 협력 실태를 질적 실증 연구 방법론으로 분석한다. 심층 인터뷰 데이터(21,011줄)를 구조화 코딩하여 장애요인, 성공요인, 정책 제언을 도출하고, 이를 기반으로 정책 보고서와 시각화를 생성한다.

## 분석 대상

- 15개 정부출연연구기관 (원자력, ICT, 표준, 해양, 바이오, 환경, 에너지, 기계, 화학, 항공, 기초과학, 핵융합, 정보, 지질 분야 + 총괄기관)
- 기관당 2-3명의 핵심 인력 인터뷰 (국제협력 담당 센터장, 실무 책임자, 연구책임자)
- 인터뷰 시간: 기관당 평균 3-4시간

## 코드북 구조

`codebook.md`에 정의된 코드 체계를 따른다:

### 주제 코드 (Thematic Codes)
1. **COLLABORATION_DRIVERS (CD_*)**: 국제협력 동인 및 목표 (9개 코드)
2. **COLLABORATION_TYPES (CT_*)**: 국제협력 유형 및 방식 (8개 코드)
3. **OBSTACLES_CHALLENGES (OBS_*)**: 장애물 및 도전과제 (24개 하위 코드)
4. **SUCCESS_FACTORS (SUC_*)**: 성공 요인 및 촉진제 (7개 코드)
5. **FAILURE_FACTORS (FAIL_*)**: 실패 요인 및 교훈 (4개 코드)
6. **POLICY_RECOMMENDATIONS (REC_*)**: 정책 요구 및 제언 (9개 코드)
7. **INSTITUTIONAL_CONTEXT (CTX_*)**: 기관/분야 맥락 (4개 코드)
8. **POLICY_EVENT_RESPONSE (PER_*)**: 특정 정책/사건 반응 (3개 코드)

### 맥락 코드 (Context/Nuance Codes)
- 각 발췌문에 1개씩 부여: PROBLEM_STATED, CAUSE_ANALYSIS, EFFECT_DESCRIBED, SOLUTION_PROPOSED, POSITIVE_EXAMPLE/FACTOR, NEGATIVE_EXAMPLE/FACTOR, FACTUAL_REPORTING, OPINION_EXPRESSED, INST_SECTOR_SPECIFIC, COMPARISON

## 에이전트의 역할

### 1단계: 인터뷰 코딩
- `raw/` 폴더의 각 인터뷰 파일을 읽고, 의미 단위(semantic unit)로 분절한다.
- 각 의미 단위에 대해 **가장 구체적인 주제 코드 1개 이상** + **맥락 코드 1개**를 부여한다.
- 코딩 결과를 `qualitative/` 폴더에 기관별로 저장한다.

### 2단계: 빈도 집계 및 패턴 분석
- 전체 코딩 결과에서 코드별 출현 빈도, 기관별 분포, 코드 간 공출현 패턴을 분석한다.
- 기관 유형별(자원 의존형, 표준 선도형, 기술 추격형) 차이를 식별한다.

### 3단계: 산출물 생성
- **정책 보고서**: 서론, 이론적 프레임워크, 현황 분석, 심층 분석(5대 구조적 문제), 정책 제언, 결론
- **시각화**: 기관별 장애요인 히트맵, 인과관계 네트워크, 정책 로드맵 타임라인
- **정책 메모**: 기관별 인터뷰 요약 + 테마 분석

## 코딩 일관성 유지 규칙

1. **가장 구체적인 코드를 우선 적용**: 예) "예산이 부족하다"는 OBS_FUNDING이 아니라 OBS_FUND_INSUFFICIENT_CORE 또는 OBS_FUND_LACK_NEW_PROJECT 등 하위 코드를 적용
2. **복수 코드 허용**: 하나의 발언이 여러 주제를 동시에 다루면 복수의 주제 코드를 부여 (단, 맥락 코드는 1개만)
3. **기관 고유 맥락 보존**: 동일한 코드라도 기관별 맥락이 다를 수 있으므로, 원문 인용을 함께 보존
4. **모호한 경우 상위 코드 사용**: 하위 코드 중 적절한 것이 없으면 상위 범주 코드를 사용

## 파일 구조

```
project_root/
├── CLAUDE.md          # 이 문서
├── codebook.md        # 코드 정의서
├── raw/               # 원본 인터뷰 녹취록 (15개 파일)
├── qualitative/       # 코딩 결과 및 분석
├── deliverables/      # 최종 보고서 및 정책 메모
├── visualizations/    # 인터랙티브 시각화 (HTML)
└── tools/             # 분석 자동화 스크립트
```

## 주의사항

- 인터뷰 참여자의 실명 및 기관 고유 명칭은 익명화하여 처리할 것
- 민감한 정책 발언은 원문 그대로 보존하되, 발언자를 식별할 수 없도록 할 것
- 코딩 결과에 대해 200개 표본 교차 검증(Cohen's Kappa >= 0.8 목표)을 수행할 것
