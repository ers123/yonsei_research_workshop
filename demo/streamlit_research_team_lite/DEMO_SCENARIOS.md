# 데모 백업 시나리오 — 나만의 연구팀 (Streamlit + Ollama)

> 워크숍 Part 2 라이브 데모용 **예비 입력 텍스트 모음**. 강사 시연 중 기본 예시(🎯 예시로 시작 버튼 → 식품영양/고령자 식품불안정)가 반응이 약하거나, 다른 전공 청중이 많을 때 즉석 대체용.
>
> 모든 시나리오는 Streamlit `app.py` 의 6개 입력 필드(프로젝트 이름 / 연구 분야 / 단계 / 키워드 / 맥락 / 추가조건) 형식에 맞춰 복붙 가능.
>
> 앱 동작 범위 주의: **웹검색 없음, 완전 오프라인.** 업로드 자료 + Ollama 로컬 모델만 사용. 실시간 최신 논문이 필요하면 산출물 내보내기 → RISS/NTIS 검색 쿼리로 우회.

---

## ⚠️ 내일 라이브 시연 — 어느 단계가 현실적인가

워크숍 당일엔 **청중이 실제 논문 PDF 를 당장 들고 있을 가능성이 낮습니다.** Lite 변형은 가상 PDF 를 자동 로드하지 않으므로, 단계별 실제 시연 가능성은 다음과 같습니다:

| 단계 | 라이브 시연 가능? | 이유 |
|---|---|---|
| **주제 탐색** | ✅ 추천 | 업로드 없이도 context 만으로 3-에이전트 파이프라인 전체 작동 |
| **선행연구 정리** | ⚠️ 제한적 | PDF 업로드 전제. 업로드 없으면 모든 출력이 `[추측]` → **교육 효과는 있지만 "정상 사용" 예시는 아님** |
| **Research Question 도출** | 🟡 가능하지만 느림 | 🧠 사고 모드 ON 권장 → 16GB 노트북에선 첫 응답 5-10분 |

**내일 권장 동선**:
- **라이브 시연**: 🎯 예시로 시작 (식품영양 · 주제 탐색) 또는 Backup 1/4 (주제 탐색 계열)
- **"업로드 없으면 왜 문제인지" 장면용**: 선행연구 정리 단계로 한 번 돌려서 **모든 출력에 `[추측]` 태그 붙는 것** 을 시각화 → "자료 필요성" 메시지 강화
- **홈스터디 안내**: 본인이 RISS/Scholar 에서 다운받은 실제 PDF 3-5개로 선행연구 정리 · RQ 도출 단계를 집에서 해보라고

## 기본 시나리오 (이미 앱에 내장) — 식품영양 · 주제 탐색

🎯 "예시로 시작" 버튼 한 번으로 6개 필드 자동 채움. `templates.py` 의 `DEMO_INPUTS` 참조.

- 프로젝트: `예시_고령자_식품불안정_석사탐색`
- 키워드: 고령자, 1인가구, 식품불안정, 영양소 섭취
- **샘플 자료**: **자동 로드 없음** (가상 논문 제공은 환각 시연과 동일하므로 제거). 본인 PDF 를 업로드하거나, 자료 없이 돌려서 `[추측]` 태깅 시각화.

## Backup 1 — 노화과학 · 주제 탐색

**프로젝트 이름**
```
노화_디지털소외_석사탐색
```
**연구 분야**: `노화과학`
**단계**: `주제 탐색`

**키워드**
```
고령자, 디지털 리터러시, 키오스크, 사회적 고립, 웰에이징
```

**맥락**
```
연세대학교 생활과학대학 노화과학 석사 1년차입니다.
65세 이상 도시 거주 노인의 디지털 소외와 일상 기능(특히 외식·금융·의료예약)
사이의 연관에 관심이 있습니다. 고령화연구패널(KLoSA) 공개데이터를 활용할 수
있고, 구청 복지관과 연계한 소규모 인터뷰(15명 내외)도 가능합니다.
양적 분석 중심이지만 질적 보완을 얹는 혼합연구를 선호합니다.
```

**추가조건**
```
코로나 이전/이후 비교는 피해주세요. 장기요양등급 수급자는 제외 모집단입니다.
6개월 내 IRB 심의까지 도달 가능한 범위로 좁혀주세요.
```

## Backup 2 — 섬유패션 · 선행연구 정리  ⚠️ 홈스터디 전용

> **내일 라이브 시연엔 비추**: PDF 업로드 전제 시나리오라 즉석 업로드할 논문이 없으면 모든 출력에 `[추측]` 태깅이 붙음. 참석자가 집에서 Google Scholar 에서 관련 논문 3-5편 다운받은 후 재현용.

**프로젝트 이름**
```
Z세대_중고패션_선행정리
```
**연구 분야**: `섬유패션`
**단계**: `선행연구 정리`

**키워드**
```
Z세대, 중고의류, 리커머스 플랫폼, 지속가능 소비, 자아표현
```

**맥락**
```
석사 2년차, 주제는 확정: "Z세대의 중고의류 구매행동과 지속가능성 인식의 관계".
국내외 최근 5년 논문 12편 정도를 1차 수집한 상태이고, 이번 단계에서는
테마 매트릭스(이론 렌즈 / 방법 / 표본 / 주요 발견)로 정리하고 싶습니다.
번역·리패브릭·빈티지 플랫폼(번개장터·Depop·Vestiaire)을 모두 포함합니다.
```

**추가조건**
```
B2B 리커머스, 명품 진품감정 논문은 제외.
이론 프레임은 가치-믿음-규범(VBN) 또는 계획된 행동이론(TPB) 계열 우선 정리.
```

> 이 시나리오로 돌릴 땐 `선행연구 정리` 체크리스트대로 PDF 3-5편 업로드 권장 → `[근거:파일명]` 태그가 풍부해져 근거 원장 시연 효과 큼.

## Backup 3 — 주거복지 · Research Question 도출

**프로젝트 이름**
```
청년1인가구_주거불안_RQlock
```
**연구 분야**: `주거복지`
**단계**: `Research Question 도출`

**키워드**
```
청년 1인가구, 주거불안정, 정신건강, 전세사기, 공공임대
```

**맥락**
```
박사 수료, 주제·선행연구 정리 완료. 연구 갭은 "청년 1인가구 주거불안정의
정신건강 경로에서 '점유형태(전세/월세/공공임대)'가 조절변수로 제대로
검토되지 않음"입니다. 활용 데이터는 청년삶실태조사 2022·2024 2개 웨이브.
SEM(구조방정식) 가능하고, 전세사기 경험 여부를 더미변수로 넣고 싶습니다.
```

**추가조건**
```
RQ는 반드시 반증가능하게 진술해주세요. 가설 수는 3개 이하.
인과추론(DiD/IV)까지 요구하는 RQ는 이번 박사논문 범위 밖이라 지양.
```

> 준호(반증가능성 공격) 장면을 강조하려면 이 시나리오 + 🧠 사고 모드 ON 권장.

## Backup 4 — 디자인 · 주제 탐색 (짧은 시간용)

**프로젝트 이름**
```
배리어프리UX_박사탐색
```
**연구 분야**: `디자인`
**단계**: `주제 탐색`

**키워드**
```
배리어프리, 저시력, 모바일 결제, UX, 접근성 가이드라인
```

**맥락**
```
디자인 전공 박사 1년차. 저시력 사용자(시력 0.3 이하, 60대+)의 간편결제 앱
사용 경험에 관심. 협력 가능한 기관: 한국시각장애인복지관. 예비 참가자
8명 확보. WCAG 2.2 / KS X 0815 를 준거로 삼고 싶습니다.
```

**추가조건**
```
완전 실명(전맹) 사용자는 이번 연구 범위 밖.
방법은 사용성 평가 + 반구조화 인터뷰 조합 선호. 아이트래킹 장비 없음.
```

> 시간 압축 버전(DEMO_SCRIPT 부록 C, 4분 압축판)에 가장 잘 어울림. 맥락이 짧아 첫 토큰이 빨리 나옴.

## 자주 나오는 질문 대응 메모

- **"웹검색도 되나요?"** → "안 됩니다. 이건 완전 로컬 앱이에요. Wi-Fi 꺼도 돌아갑니다. 최신 논문이 필요하면 여기서 **RISS 검색 쿼리**를 뽑아서 직접 검색하는 흐름입니다." (→ 산출물 내보내기 → `RISS/NTIS 검색 쿼리 (.txt)` 시연)
- **"ChatGPT 로 하면 더 좋지 않나요?"** → "프런티어 모델을 대체하려는 게 아니라, **반복 · 프라이버시 · 비용** 세 축에서만 로컬로 옮기는 컴패니언입니다. IRB 전 미발표 원고를 20번 돌려야 할 때 결정적 차이가 납니다."
- **"우리 학교 데이터 줘도 되나요?"** → "네, 업로드한 파일은 `.sessions/` 와 로컬 메모리에만 남습니다. 외부로 전송되는 건 Ollama 로컬 서버로 가는 호출뿐입니다."
