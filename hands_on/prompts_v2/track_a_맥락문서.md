# Track A — 프로젝트 맥락 문서 만들기

**시간**: 45분  
**목표**: 본인 연구에 맞는 CLAUDE.md를 작성하고, 맥락 문서가 있을 때 / 없을 때 차이를 직접 체감한다.

---

## 왜 이걸 하는가

채팅창에서 "서론 써줘"라고 할 때마다 배경부터 다시 설명하고 있다면 — 그 시간이 누적되고 있는 것.
맥락 문서 한 장이면 모든 대화가 그 문서를 기준으로 시작한다.

**30분 투자 → 이후 30시간 절약.** KAIST AAR에서 이 주장에 "굳이?"라는 반응이 나왔지만, 3주 뒤 다시 연락이 왔다: "진짜 다르더라."

---

## Step 0 — 준비 (2분)

본인이 지금 가장 많이 사용하는 AI 도구(Claude, ChatGPT 등)를 열어두세요.
아래 둘 다 사용할 겁니다:
1. **맥락 문서 없이** 질문하는 세션 (A 세션)
2. **맥락 문서 넣고** 같은 질문하는 세션 (B 세션)

---

## Step 1 — 5분 작성: 빈칸 채우기

아래 템플릿을 복사해서 빈칸을 채우세요. 완벽하지 않아도 됩니다.

```markdown
# 프로젝트 맥락 문서 (CLAUDE.md)

## 프로젝트
- 주제: ________
- 연구 질문: ________
- 대상/자료: ________
- 방법론: ________
- 현재 단계: [문헌검토 / 자료수집 / 분석 / 작성] 중 ________

## 독자
- 이 연구를 읽을 사람: ________ (학술지 심사위원 / 지도교수 / 정책 관계자)
- 독자가 이 분야에 대해 아는 수준: [전문가 / 비전공자 / 혼합]

## 출력 규칙
- 문체: [학술체 ~이다 / 보고서체 ~합니다 / 구어체]
- 인용 형식: [APA 7th / Chicago / 한국어 학회지 형식]
- 언어: [한국어 / 영어 / 혼용]
- 분량 제한: [문단당 4-6문장 / 전체 3000자 이내 등]

## 금기
- 확인 안 된 논문은 만들지 말 것. 모르면 "확인 필요" 표기.
- 내 연구 질문을 임의로 바꾸지 말 것.
- ________ (본인만의 금기 추가)

## 핵심 선행 연구
- ________ (저자, 연도) — 한 줄 요약
- ________ (저자, 연도) — 한 줄 요약
- ________ (아직 없으면 "탐색 중" OK)
```

---

## Step 2 — 비교 실험 (15분)

### A 세션: 맥락 없이 질문

새 채팅을 열고 이렇게만 질문하세요:

```
내 연구 주제로 서론 도입부를 2문단 써줘.
주제: [본인 주제 한 줄]
```

결과를 저장.

### B 세션: 맥락 문서 넣고 같은 질문

새 채팅을 열고:

```
아래는 내 프로젝트 맥락 문서입니다. 모든 대화에서 이 문서를 기준으로 답해주세요.

---
(Step 1에서 작성한 전체 내용 붙여넣기)
---

위 맥락을 바탕으로 서론 도입부를 2문단 써줘.
```

결과를 저장.

### 비교

A와 B를 나란히 놓고 다음을 체크하세요:

| 항목 | A (맥락 없음) | B (맥락 있음) |
|---|---|---|
| 문체가 내 연구에 맞나 | | |
| 인용 형식이 맞나 | | |
| 독자 수준에 맞나 | | |
| 다시 요청 없이 쓸 수 있는 수준인가 | | |

→ 차이가 느껴지면 성공. 안 느껴지면 Step 1의 맥락 문서가 너무 비어있는 것 — 돌아가서 채우세요.

---

## Step 3 — 반복성 테스트 (10분)

같은 맥락 문서를 다른 AI 도구에도 넣어보세요 (Claude에서 ChatGPT로, 또는 반대로).

```
아래 프로젝트 맥락 문서를 읽고, 서론 도입부 2문단을 써줘.
(맥락 문서 붙여넣기)
```

**결과가 비슷하면**: 맥락 문서가 명확하다는 증거.
**결과가 완전히 다르면**: 맥락 문서에 모호한 부분이 있다 — 구체적으로 수정.

---

## Step 4 — Claude Code에서 자동 로드 해보기 (선택, 10분)

Claude Code 사용자라면:

```bash
# 1. 프로젝트 폴더에 CLAUDE.md 저장
cp CLAUDE.md ~/my_research_project/CLAUDE.md

# 2. 그 폴더에서 Claude Code 실행
cd ~/my_research_project
claude

# 3. Claude Code가 CLAUDE.md를 자동으로 읽는다.
#    별도 붙여넣기 없이 바로 맥락이 적용됨.
```

이렇게 하면 매번 복붙할 필요가 없어집니다. 파일이 프로젝트 루트에 있으면 자동 참조.

---

## Step 5 — 마무리: AGENTS.md도 만들어 보기 (남는 시간에)

CLAUDE.md가 "이 프로젝트가 뭔가"라면, AGENTS.md는 "어떻게 일할 것인가".

```markdown
# AGENTS.md

## 일반 원칙
- 애매하면 가정하지 말고 물어라
- 원자료(raw/)는 절대 수정하지 않는다
- 모든 인용에 출처를 붙인다 — 없으면 "확인 필요"

## 파일 규칙
- 기존 파일 덮어쓰기 금지 (백업: file_YYYYMMDD.md)
- 대용량 파일은 헤더/샘플만 먼저 확인

## AI 사용 공개
- Where: 어느 부분에 사용했는가
- Why: 왜 사용했는가
- How much: 얼마나 사용했는가
```

- [ ] CLAUDE.md 빈칸을 80% 이상 채웠다
- [ ] A/B 비교에서 체감할 수 있는 차이가 있었다
- [ ] 다른 AI 도구에서도 비슷한 결과가 나왔다
- [ ] (선택) Claude Code에서 자동 로드를 확인했다

## 함정 피하기

- **맥락 문서에 모든 걸 넣으려 하지 마세요.** 한 페이지. 자주 반복하는 것만. 부가 정보는 링크로.
- **문서가 실제와 다르면 바로 고쳐야 합니다.** 어제의 결정과 오늘의 결정이 다르면, 에이전트는 어제 기준으로 답합니다.
- **"나한테만 통하는 문서"도 가치 있습니다.** 동료에게 넘기게 될 때 다듬으면 됩니다.

## 올해 달라진 것: Prompt → Context → Harness

| 작년 (Prompt) | 올해 (Harness) |
|---|---|
| 매번 배경을 복붙 | CLAUDE.md가 자동 로드 |
| 도구 바꾸면 처음부터 | 같은 문서를 Claude/ChatGPT/Gemini에 공유 |
| 결과가 산발적 | 출력 규칙·금기가 일관성 보장 |
| 내가 다 검증 | AGENTS.md에 검증 규칙까지 포함 |

**맥락 문서 = harness의 가장 기본 단위.** Track C에서 이 위에 교차검증까지 쌓습니다.
