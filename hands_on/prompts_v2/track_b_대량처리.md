# Track B — 대량 자료 처리

**시간**: 45분  
**목표**: 강사가 준비한 샘플 PDF 10건으로 "추출 → 구조화 → 조회"를 한 사이클 돌아본다.

---

## 왜 이걸 하는가

선행 연구 40편을 3일 안에 훑어야 한다. 인터뷰 전사 20건에서 공통 주제를 뽑아야 한다.
PDF를 하나씩 열어서 ChatGPT에 복붙하면 — 맥락이 매번 끊기고, 출처가 섞이고, 검증이 안 된다.

**핵심**: 추출 단계를 분리한다 + 출처를 남긴다 + 에이전트가 반복하게 한다.

---

## 준비물

- 강사 제공 샘플 PDF 10건 (`hands_on/sample_data/` 폴더 — 당일 배포)
- 또는 본인이 가져온 PDF (논문, 보고서, 전사록 등)
- Claude 또는 ChatGPT (무료 OK)
- (선택) Claude Code 터미널

---

## Step 1 — 추출: PDF에서 텍스트 꺼내기 (10분)

### 방법 1: 복붙 (PDF 1–3편, 본문만 필요할 때)

PDF 열고 → 전체 선택 → 복사 → 텍스트 파일에 붙여넣기.
**한계**: 표가 깨진다. 3편 넘으면 시간 낭비.

### 방법 2: Claude에 직접 올리기 (가장 실용적)

Claude에 PDF를 드래그해서 올리고:

```
이 PDF의 내용을 다음 형식으로 정리해줘:

## 메타
- 제목:
- 저자:
- 연도:
- 학술지:
- DOI (있으면):

## 핵심 요약 (200자 이내)

## 방법론 한 줄

## 주요 발견 3개 (각 한 줄)

## 직접 인용할 만한 문장 (있으면, 페이지 표시)
```

→ 이걸 10번 반복하면 **10편이 동일 포맷으로 정리된다.** 이게 핵심.

### 방법 3: Claude Code로 배치 처리 (빠름, 추천)

```bash
# 프로젝트 폴더에서
claude

# Claude Code에게:
hands_on/sample_data/ 폴더에 PDF 10개가 있어.
각 PDF를 읽고, 아래 형식으로 output/summaries/ 폴더에 
파일명_summary.md로 저장해줘.

형식:
## 메타
- 제목:
- 저자:
- 연도:
- 학술지:

## 핵심 요약 (200자 이내)
## 방법론 한 줄  
## 주요 발견 3개
## 인용할 만한 문장 (페이지 표시)
```

→ Claude Code가 10개를 순서대로 돌며 각각 파일로 저장한다.
→ 수동 복붙 대비 10배 빠르고, 포맷이 통일된다.

---

## Step 2 — 구조화: 분류 체계 만들기 (15분)

Step 1에서 나온 10개 요약을 가지고.

### 2-1. 먼저 본인이 분류 축을 정한다 (3분)

```markdown
# 내 분류 체계

목적: 10편 문헌에서 연구 공백(gap)을 찾기 위해

분류 축:
- 축 A: 주제 영역 [예: 식습관 / 운동 / 수면 / 복합]
- 축 B: 방법론 [양적 / 질적 / 혼합 / 리뷰]
- 축 C: 대상 [아동 / 청소년 / 성인 / 노인]

배치 규칙:
- 여러 주제면 → 가장 비중 큰 것 1개
- 방법론이 애매하면 → "혼합"으로, 괄호에 주방법 표기
```

### 2-2. 에이전트에게 분류 시키기

```
아래는 내 분류 체계와 10편 문헌 요약이야.

[분류 체계]
(위에서 정한 축 + 규칙 복사)

[문헌 요약]
(Step 1의 결과 전체 복사 — 또는 파일 경로 지정)

[요청]
1. 각 문헌을 축 A/B/C에 배치해줘
2. 표로 만들어줘 (컬럼: 제목, 주제영역, 방법론, 대상, 배치 근거 한 줄)
3. 비어 있는 셀 조합 (= 연구 공백) 상위 3개 알려줘
4. 배치가 애매했던 문헌과 그 이유도 표시
```

### 2-3. 교차 검증

같은 분류 체계 + 같은 자료를 다른 AI에도 넣어 본다.
- 분류 결과가 일치 → 체계가 명확
- 분류가 엇갈림 → 축이나 규칙에 모호함 → 수정 후 재시도

---

## Step 3 — 조회: 분류 결과로 질문하기 (15분)

구조화된 자료가 있으면 이제 "조회"가 가능하다.

### 질문 예시 (그대로 복붙해서 써보세요)

```
위 분류 결과를 바탕으로 다음에 답해줘:

1. 양적 연구가 가장 많은 주제 영역은? 왜 그렇다고 보나?
2. 질적 연구가 부족한 조합은? 그 이유를 기존 문헌에서 추론할 수 있나?
3. 내 연구 질문([본인 RQ 삽입])에 가장 직접적으로 관련된 문헌 3편은?
   각각 어떤 점에서 관련되고, 어떤 점에서 차이가 있나?
4. 이 10편만으로 볼 때, 가장 분명한 연구 공백(gap)은?
   그 공백을 메우려면 어떤 방법론이 적합할까?

모든 답에 근거가 된 문헌 번호/저자를 명시해줘.
추론이면 "추론:"이라고 표시. 자료에 없으면 "해당 자료 없음"이라고.
```

### 출처 매핑 확인

에이전트의 답변에서 인용된 문헌이 실제로 그 내용을 담고 있는지 **최소 2건은 직접 확인하세요.**
→ 안 하면 검증부채(Verification Debt)가 쌓인다. 나중에 한꺼번에 터진다.

---

## Step 4 — 저장 & 재사용 (5분)

모든 결과를 파일로 남깁니다:

```
output/
├── summaries/          ← Step 1 결과 (PDF별 요약)
├── classification.md   ← Step 2 결과 (분류표 + 공백)
└── analysis.md         ← Step 3 결과 (질문-답변 + 출처)
```

**핵심**: 이 파일들이 남아 있으면 — 다음에 PDF 10개가 더 들어와도 같은 체계로 이어서 분류할 수 있다.
모델을 바꿔도 이 흐름은 유지된다. 이게 Harness Engineering의 실제 형태.

---

## 체크포인트

- [ ] PDF 최소 5편의 요약이 동일 포맷으로 정리됨
- [ ] 분류 축을 본인이 직접 설계함 (에이전트가 임의로 만든 게 아님)
- [ ] 분류표에서 연구 공백 최소 2개를 식별함
- [ ] 조회 결과에서 출처가 실제 문헌과 매핑됨 (최소 2건 확인)
- [ ] 결과가 파일로 저장됨

---

## 함정 피하기

- **PDF를 하나씩 별도 세션에서 묻지 마세요.** 요약 포맷을 통일해야 나중에 분류가 된다.
- **"이 자료 구조화해줘"라고 하지 마세요.** 에이전트가 임의 기준으로 분류한다. 재현 불가. 축을 본인이 먼저 정해야.
- **한 번에 50편 넣지 마세요.** 10–15편씩 배치. 에이전트 맥락이 넘치면 앞을 잊는다.
- **출처 매핑을 건너뛰지 마세요.** "에이전트가 표를 예쁘게 만들어줬으니 맞겠지" → 틀린 케이스를 여러 번 봤다.

## 올해 달라진 것: 왜 "harness"인가

| 작년 (Tool Hopping) | 올해 (Harness) |
|---|---|
| PDF마다 새 채팅에서 복붙 | Claude Code가 폴더째 배치 처리 |
| 분류 기준을 AI가 임의로 | 내가 축을 설계, AI가 실행 |
| 결과가 채팅에만 존재 | 파일로 저장 → 재사용 가능 |
| 검증? 눈으로 | 교차검증 가능 (Track C 참조) |

**파이프라인이 파일로 남는다 = 다음에 PDF 10개가 더 와도 같은 체계로 이어간다.**
이게 Harness Engineering의 실제 형태.

---

## 팁: `/loop`으로 대량 처리 자동화

PDF가 많을 때 Claude Code에서:

```
/loop sample_data/ 폴더의 PDF를 하나씩 읽고 
output/summaries/에 동일 포맷으로 저장해줘.
이미 처리된 파일은 건너뛰어.
```

사람이 10번 반복할 걸 시스템이 자동으로.
