# Track B — 대량 자료 처리

**시간**: 30-35분
**목표**: 대용량 PDF를 "중간 산출물(AI-friendly markdown)"로 변환해 **재사용 가능한 자산**으로 만드는 법을 체험한다. Naive 접근이 어디까지 잘 되고, 어디서 무너지는지를 직접 본다.

**실습 자료**: Stanford HAI의 *AI Index Report 2026* (423페이지, 약 37MB 공개 보고서)
→ `hands_on/sample_data/ai_index/ai_index_report_2026.pdf`

---

## 왜 이걸 하는가

선행 연구 40편, 인터뷰 전사 20건, 400페이지 정책 리포트. PDF를 하나씩 열어서 챗봇에 복붙하면 — 매번 처음부터. 결과는 채팅창에만 남고, 출처 추적은 흐려진다.

**핵심 전환**: 원본 PDF를 한 번만 읽고, AI가 잘 다룰 수 있는 **중간 산출물**(마크다운)로 추출해 둔다. 이후 모든 질문은 이 중간 산출물에 던진다. 한 번의 추출 비용으로 N번의 질문·검증·재활용이 가능해진다.

> 오늘 쓸 표현: **"PDF는 소비 대상이 아니라 자산이다."**
> Naive 접근 = 1회 소비. Harness 접근 = N회 재사용.

## Part 1 — Naive로 시작: 되는 것과 안 되는 것 (7분)

### 1-2. 난이도별 질문 3개 던지기

업로드가 성공했다면, 아래 3개 질문을 **순서대로** 던져보세요. 각 답변의 정확도·구체성·출처 표시를 메모하세요.

**질문 A — 쉬움 (본문과 차트에 수치가 반복되는 유형)**
```
AI Index 2026 리포트에서 2025년 미국, 중국, 한국의 notable AI 모델 개발 수를 정확히 알려줘.
```
→ 본문에 "50, 30, 5"로 명시. Figure 1.1.1에도 동일. **잘 될 확률 높음.**

**질문 B — 중간 (본문에만 언급, 차트는 없음)**
```
리포트에서 한국이 세계 1위인 항목이 무엇인지, 왜 그런지 근거와 함께 알려줘.
페이지 번호도 같이.
```
→ "South Korea leads in AI patents per capita" (Top Takeaways + Chapter 1 highlights). **페이지 인용이 맞는지** 확인.

**질문 C — 어려움 (시각 정보에만 있는 데이터)**
```
Figure 1.1.3의 세계지도에서 '181-630' 구간(최상위)에 속한 국가를 모두 알려줘.
추측하지 말고 지도에서 실제로 읽은 것만.
```
→ 이 정보는 **색상 음영으로만** 표현됨. 텍스트 레이어에 없음. **환각 또는 '확인 불가' 가 정답.**

### 1-3. 관찰 포인트 — 체크리스트

| 질문 | 답이 정확했나 | 페이지 인용이 맞았나 | 추측·환각이 섞였나 |
|---|---|---|---|
| A (쉬움) | | | |
| B (중간) | | | |
| C (어려움) | | | |

**보통 나오는 패턴**:
- A: 대부분 성공 (본문 + 차트 중복 덕분)
- B: 성공하지만 페이지 인용이 모호해질 수 있음
- C: **환각 발생 확률 높음** — AI가 그럴듯한 국가 리스트를 지어내는 경우 다수

> **여기서 멈추고 생각하기**: Naive가 실패하는 건 "Claude가 멍청해서"가 아니라, **이미지(맵·중간 차트값)를 텍스트 레이어로 추출할 수 없기 때문**. 그러니 텍스트 레이어를 먼저 정비해둬야 한다 → Part 2.

## Part 2 — 중간 산출물 만들기 (10분)

> **목적**: PDF를 한 번만 읽고, AI가 잘 다룰 수 있는 **마크다운**으로 추출해서 파일로 남긴다.

### 2-1. 추출이란 무엇인가

"추출"은 번역이 아니라 **구조화**입니다. 목표 형태:

```markdown
# AI Index Report 2026 — 추출본

## Top Takeaways
1. AI capability is not plateauing...
2. The U.S.-China AI model performance gap has effectively closed...

## Chapter 1. Research and Development

### 1.1 Notable AI Models

**By National Affiliation (2025)**

| 국가 | notable AI 모델 수 |
|---|---|
| United States | 50 |
| China | 30 |
| South Korea | 5 |
| Canada | 1 |
| France | 1 |

[Figure 1.1.3: 2003-25 누적 notable AI 모델 수 세계지도.
 최상위 구간(181-630): United States, China.
 상위 구간(61-180): Canada, Germany, UK, France, Japan...
 (나머지 기재)]
```

핵심 규칙:
- 본문은 평문 마크다운
- 표는 마크다운 표로 재구성
- 차트/그림은 `[Figure X.Y: 설명 + 주요 수치]` 캡션으로 대체
- 출처 페이지를 각 섹션 끝에 `(p. 13-15)` 같은 식으로 명시

### 3-4. 비교표 작성

| 질문 | Part 1 (Naive) | Part 3 (Harness) | 차이 |

```
추출 비용 (1회): ~10분 + AI 토큰 1회
Naive 방식 (N회 질문): 37MB × N번 재처리 (속도·토큰·실패 리스크)
Harness 방식 (N회 질문): 수백KB × N번 (빠름·정확)

손익분기: N=2 정도부터 Harness 우세. N=5+ 압도적.
```

---

## 체크포인트

- [ ] Part 1에서 Naive가 **어디서 잘 되고 어디서 실패하는지** 직접 확인함
- [ ] 중간 산출물(`ai_index_extracted.md`) 파일이 생성됨 (직접 추출 또는 백업)
- [ ] Part 3에서 같은 질문의 결과가 **더 정확해지거나 더 정직해짐**을 확인
- [ ] 같은 추출본으로 최소 2개의 다른 질문에 답변을 받음 (재사용성)
- [ ] 본인 연구 PDF에 이 흐름을 적용할 수 있을 것 같은지 판단

## 함정 피하기

- **업로드 한 번으로 모든 걸 기대하지 마세요.** 37MB PDF는 챗봇의 가장 만만한 입구가 아닙니다. 실패가 정상.
- **"추출"을 단순 복사라고 생각하지 마세요.** 표는 표로, 그림은 캡션으로 — **구조화**가 핵심입니다.
- **추출본을 만들고 끝내지 마세요.** 자산은 재사용할 때 가치가 생깁니다. 최소 2-3개 질문을 추출본에 다시 던져보세요.
- **환각을 발견하면 기록하세요.** "어떤 유형의 질문에서 AI가 지어냈는지" 패턴이 쌓이면 본인만의 검증 체크리스트가 됩니다.

---

## 올해 달라진 것: "PDF는 소비 대상이 아니라 자산이다"

| 작년 (Tool Hopping) | 올해 (Harness) |
|---|---|
| PDF마다 새 채팅에서 복붙 | 한 번 추출, 마크다운으로 보관 |
| 맵·차트 정보는 포기 | 캡션으로 재구성해서 보존 |
| 페이지 인용? 운에 맡김 | 추출 단계에서 페이지 명시 |
| 결과가 채팅에만 존재 | `.md` 파일로 저장, 동료에게 공유 |
| 질문마다 37MB 재처리 | 수백KB 마크다운에 N번 질의 |

**파이프라인이 파일로 남는다 = 이 흐름을 다른 PDF에도 그대로 재사용.**
이게 Harness Engineering의 실제 형태.

## (심화 1) 본인 PDF에 적용

본인 연구 자료(PDF)가 있다면 방금 배운 흐름을 그대로 적용:

```
[본인 PDF] → Claude Code로 추출 → [my_paper_extracted.md] → 질의

프롬프트 템플릿:
"이 PDF를 AI-friendly 마크다운으로 추출해줘. 본문은 평문,
 표는 마크다운 표, 그림은 [Figure X: 설명] 캡션, 섹션 끝에 페이지."
```

> **⚠️ 스캔본 주의**: 본인 PDF가 스캔된 이미지라면 텍스트 레이어가 없어 추출이 안 됩니다. 이 경우 먼저 OCR(광학문자인식) 처리 필요 — macOS 미리보기 기본 OCR, Adobe Acrobat Pro, 또는 `ocrmypdf` (명령줄) 활용.

---

## (심화 2) `/loop`으로 대량 문헌 자동 추출

문헌이 여러 편이면 Claude Code에서:

```
/loop my_pdfs/ 폴더의 PDF를 하나씩 읽고
output/extracted/에 같은 포맷의 마크다운으로 저장해줘.
이미 처리된 파일은 건너뛰어.
```

사람이 N번 반복할 걸 시스템이 자동으로. N=40편의 리뷰 논문 → 1시간 이내 전부 구조화된 마크다운으로.

## (심화 3) 분류·비교 — sample_data/ 10건 활용

Track B의 기본은 "대용량 1건"이지만, "여러 건 횡단 분류"도 자주 필요합니다.
`hands_on/sample_data/sample_01~10.txt` (가상의 연구논문 10건)로 연습 가능:

```
sample_data/ 폴더의 10개 파일을 읽고 다음 축으로 분류해줘:
- 축 A: 주제 영역 (식품영양 / 노화 / 주거 / 섬유 / 디자인)
- 축 B: 방법론 (양적 / 질적 / 혼합 / 리뷰)
- 축 C: 대상 (아동 / 청소년 / 성인 / 노인 / 전체)

표로 정리하고, 빈 칸 조합(=연구 갭) 상위 3개를 알려줘.
```

이 흐름의 상세는 **자습용 예제**로 `hands_on/sample_data/README.md` 참조.
