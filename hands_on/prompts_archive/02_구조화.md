# 실습 2: 구조화

**핵심 원칙:** "목차를 만들어"가 아니라 **"목차를 만드는 체계를 설계해"**.

신정규 (Lablup, 2026) 방식으로 표현하면: "만드는 놈을 만들어달라."

---

## 왜 이게 중요한가?

AI에게 "이 자료의 목차를 짜줘"라고 하면:
- 매번 다른 결과가 나옵니다 (재현성 0).
- 본인 머릿속 판단 기준이 드러나지 않습니다.
- 나중에 다른 자료가 들어오면 또 새로 해야 합니다.

"목차를 만드는 기준을 먼저 설계한다" → 재사용 가능, 검증 가능, 확장 가능.

---

## 쓸 수 있는 구조 도구들

연구 유형에 따라 취사 선택:

| 작업 | 적합한 체계화 도구 |
|---|---|
| 문헌 정리 | 범주표 (분야 × 방법론 × 연도) |
| 질적 자료 분석 | 코드북 (주제 코드 + 맥락 코드) |
| 혼합 연구 | 분석 프레임 (RQ → 가설 → 변인) |
| 정책/제안 | 논리모델 (투입 → 활동 → 산출 → 성과) |

**이번 실습은 "문헌 정리 범주표" 기준**으로 진행. 본인 과제에 맞춰 응용하세요.

---

## 단계별 진행

### Step 1: 체계 설계 (사람이 주도)

에이전트가 아닌 **본인이 먼저** 2-3분 생각해서 작성:

```markdown
# 내 분류 체계

목적: [예 - "수면 관련 문헌을 학문 분야별로 묶어 갭을 찾기 위해"]

분류 축:
- 축 A: 학문 분야 [공중보건 / 심리학 / 교육학 / 가정학]
- 축 B: 방법론 [양적 / 질적 / 혼합 / 리뷰]
- 축 C: 대상 연령 [유아 / 청소년 / 성인 / 노인]

배치 규칙:
- 분야가 모호하면 제1저자 소속 기준
- 혼합 연구는 "혼합"으로 분류, 주방법 괄호로 표시
- 대상이 여러 연령이면 주 대상 1개 선택
```

### Step 2: 에이전트에게 체계 + 자료 주입

```
[역할]
너는 내 문헌 정리 보조자야.

[체계]
(위 Step 1 결과를 그대로 복사)

[자료]
실습 1에서 얻은 문헌 10개를 제목·저자·초록·연도와 함께 붙여넣기.

[요청]
1. 각 문헌을 축 A/B/C에 배치
2. 결과를 표로 제시 (컬럼: 제목, 분야, 방법론, 대상, 배치 근거)
3. 셀이 비어있는 조합 (즉, 미답 공백) 상위 3개 제시
4. 애매했던 케이스와 그 이유 명시
```

### Step 3: 반복 가능성 테스트

같은 프롬프트를 Claude와 ChatGPT에 각각 실행.
- 비슷한 분류 → 체계가 명확
- 엇갈림 → 체계에 모호함 있음 → Step 1로 돌아가 규칙 추가

### Step 4: CLAUDE.md에 체계 저장

검증된 체계를 `CLAUDE.md`에 저장. 다음에 새 자료가 들어와도 재사용.

## 심화 — 질적 연구라면

인터뷰/FGI 자료를 분석 중이라면 이 단계에서 **코드북**을 만드세요.
구조는 같습니다: "체계 설계 → 에이전트에게 주입 → 반복성 확인 → CLAUDE.md 저장".

코드북 예시는 `demo/qualitative_research/`에 실제 사례가 있습니다.

---

## 함정 피하기

- ❌ **"이 자료 구조화해줘"** — 에이전트가 임의로 분류. 재현 불가.
- ✅ **"이 체계대로 분류해줘"** — 결과 검증 가능.
- ❌ **한 번에 50개 자료** — 에이전트 context overload. 10-20개씩 배치.
- ✅ **점진적 분류** — 10개 분류 → 체계 점검 → 다음 10개.
