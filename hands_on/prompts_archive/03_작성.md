# 실습 3: 작성

**핵심 원칙:** AI에게 **"어떻게 쓸지의 구조를 먼저 명시"**하고 분량/어조/인용 형식까지 제약을 걸기.

Anthropic (2025) 조사: "해줘" vs "어떻게 생각해?" → 후자가 평균 **52% 더 높은 품질**.

---

## 함정: "서론을 써줘"의 위험

- 주제별 길이가 제멋대로
- 같은 주장이 반복됨
- 인용 위치가 엉뚱함
- 본인 어조와 다름

→ 이 단계를 에이전트에게 "맥락 없이" 맡기면 시간 낭비 + 삭제 작업만 늘어납니다.

---

## 권장 구조

### 1단계: 논리 뼈대 먼저 (사람이 작성, 10분)

```markdown
# [논문 제목 draft]

## 서론
- 문단 1: 현상 기술 [어떤 사회 현상이 일어나고 있나, 1문단]
- 문단 2: 선행 연구의 접근 [기존에 어떻게 봐왔나, 2문단]
- 문단 3: 남은 공백 [무엇이 해결 안 됐나, 1문단]
- 문단 4: 본 연구의 기여 [뭘 할 건가, 1문단]
- 각 문단 마지막에 (저자, 연도) 형식으로 인용 붙여줘

## 방법
...

## 결과
...
```

### 2단계: 에이전트에게 체계 + 자료 + 제약 전달

```
[역할]
너는 본인의 공저자야. 초고 작성을 도와줘.

[맥락]
(CLAUDE.md 내용 붙여넣기 — 주제, 대상, 방법)

[자료]
(실습 1-2에서 확보/정리한 문헌 요약)

[구조]
(위 1단계 뼈대 붙여넣기)

[제약]
- 문단당 4-6문장
- 단문 위주. 20자 넘는 부사구 금지.
- 학술 문체. "~이다/~한다" 기본.
- 모든 주장에 (저자, 연도) 형식 인용.
- 본인이 쓸 초안이지 최종본 아님. 과감하되 겸손.

[주의]
- 자료에 없는 내용은 만들지 말 것. 빈 정보는 [확인 필요]로 표시.
- 자신 없으면 답변 말고 질문하는 게 낫다.
```

### 3단계: 교차 확인 후 본인이 통합

1. Claude와 ChatGPT로 같은 프롬프트 두 번 실행
2. 두 결과의 **공통 뼈대**는 유지
3. **차이 부분**은 원문과 대조해 본인이 선택/재작성
4. AI 생성 문장 그대로 두지 말고 **본인 목소리로 덧쓰기**

**이 3단계가 Verification Debt (Kwon 2026)를 방지하는 최소 장치입니다.**

---

## 윤리적 공개 (Stapleton, 2026)

논문 초고/감사의 말/부록에 반드시 기록:

```
AI 도구 사용 공개:
- Where (어디에): 초록 작성, 선행연구 요약, 1차 문단 생성
- Why (왜): 구조화된 초안 작성 가속
- How much (얼마나): 전체 텍스트의 약 40%는 AI 초안 후 저자가 재작성
- Tools: Claude Opus 4.5, ChatGPT-5
```

---

## 특별 팁: Long document 다룰 때

한 번에 논문 전체를 쓰지 마세요. 섹션별로 분할:

1. 서론 → 저장
2. 방법 → 저장
3. 결과 → 저장
4. 통합 단계에서 "이 4섹션을 하나로 합치고 중복 제거, 논리 흐름 점검" 요청

각 섹션마다 CLAUDE.md를 참조해 맥락 유지.
