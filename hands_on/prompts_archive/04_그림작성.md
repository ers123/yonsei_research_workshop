## 경로 A — ChatGPT/Gemini 직접

### 프롬프트 템플릿

```
[역할]
너는 학술 figure 제작자야.

[목표]
(예: "본 논문의 연구 흐름을 독자가 한 눈에 이해할 수 있는 flowchart")

[스타일]
- 매체: 학술 저널 [journal 이름 또는 "일반적인 social science 저널"]
- 색상: 흑백 또는 단색 + 포인트 1색 (저널 인쇄 대응)
- 폰트: sans-serif, 10pt 이상
- 비율: 가로형 (2열 배치가 일반적)

[요소]
- 주요 블록: [A, B, C, D — 각각 무엇을 뜻하는지]
- 흐름: [A → B → (분기) C/D → 통합]
- 라벨: [각 블록의 한 줄 설명]
- 범례: [약어 설명 필요시]

[제약]
- 1장에 다 들어가야 함
- 저널 인쇄물(흑백 B5 기준)에서 가독 확보
- 저자 이름/논문 제목은 넣지 말 것

[포맷]
PNG 또는 SVG로 출력. 벡터면 더 좋음.
```

### 반복 개선

1차 결과물은 대개 수정 필요. 다음 질문으로 다듬기:
- "A 블록의 글씨가 너무 작다. 두 배로."
- "C와 D 사이 화살표가 어느 방향인지 불분명."
- "색상을 브랜드 파랑 대신 저널 표준 회색으로."

## 경로 B — PaperVizAgent (Google Research)

### 설치

```bash
git clone https://github.com/google-research/papervizagent
cd papervizagent
pip install -r requirements.txt
```

### 실행

```bash
python -m papervizagent --paper your_paper.pdf --figure_spec spec.yaml
```

`spec.yaml`에 원하는 figure 유형과 강조점을 기술:

```yaml
figure_type: conceptual_framework
emphasis: temporal_flow
style: nature_journal
caption: "Proposed research workflow"
```

### 특징

- 5개 sub-agent가 협력 (Retriever → Planner → Stylist → Visualizer → Critic)
- 20개 후보 동시 생성 → 자동 평가 → 상위 선택
- 2K/4K 해상도 출력
- 재현 가능 (같은 spec 파일로 재실행)

Chen, S. et al. (2026) 논문 참조 / github.com/google-research/papervizagent

## 경로 C — 코드로 직접 (matplotlib)

통계 결과 figure는 직접 그리는 게 최종적으로 제일 빠릅니다.

### 에이전트에게 스크립트 작성 의뢰

```
[데이터]
CSV 파일: data/interview_counts.csv (7개 참가자 × 8개 주제 코드)

[요청 figure]
heatmap — 셀 값은 언급 빈도(0-4). 
행: 참가자 (P01-P07).
열: 주제 코드 (이름은 한국어).
색상: 흰색→파랑 그라데이션.
높은 값에 숫자 라벨.

[출력]
Python matplotlib 스크립트.
실행하면 viz_heatmap.png 저장.
스크립트 자체에 주석으로 데이터 구조 설명.
```
