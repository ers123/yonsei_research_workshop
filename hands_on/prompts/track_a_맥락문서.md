# Track A — 프로젝트 맥락 문서 만들기

**목표**: 본인 연구에 맞는 `CLAUDE.md`를 한 페이지로 작성하고, **맥락 유/무의 차이**를 같은 질문으로 직접 체감한다.

**산출물**: `CLAUDE.md` 파일 1개 + A/B 비교 메모

> 이 트랙은 Part 3 실습의 **첫 번째 필수 트랙**입니다. 여기서 만든 CLAUDE.md는 본인 연구의 "시작 파일"로 계속 재사용됩니다. Track B에서도 같은 개념(맥락을 파일로 남기기)을 적용합니다.

---

## 왜 이걸 하는가

채팅창에서 "서론 써줘"라고 할 때마다 배경부터 다시 설명하고 있다면 — 그 시간이 누적되고 있는 것. 맥락 문서 한 장이면 모든 대화가 그 문서를 기준으로 시작한다.

**30분 투자 → 이후 30시간 절약.** 한 사후 회의에서 "굳이?"라는 반응이 나왔지만, 3주 뒤 다시 연락이 왔다: "진짜 다르더라."

## Step 0 — 준비 

### 0-1. 본인이 자주 쓰는 AI 도구 열기

Claude (claude.ai) 또는 ChatGPT (chat.openai.com). 무료 계정도 OK. 브라우저에서 **로그인 완료 상태**인지 확인.

### 0-2. 텍스트 편집기 준비

마크다운 파일(`.md`)을 저장할 편집기가 필요합니다. 아래 중 **하나** 있으면 됩니다:

| OS | 기본 사용 가능 | 권장 |
|---|---|---|
| **Windows** | 메모장 (주의: 확장자 설정 필요) | VS Code (https://code.visualstudio.com) |
| **macOS** | 텍스트편집기 (포맷 → "일반 텍스트"로 변환 필요) | VS Code |
| **공통** | — | Cursor, Obsidian, Sublime Text 등 뭐든 OK |

> **VS Code 빠른 설치**
> - Windows PowerShell: `winget install Microsoft.VisualStudioCode`

> - 또는 위 링크에서 installer 다운로드

## Step 1 — CLAUDE.md 작성 + 파일로 저장 

### 1-1. 템플릿 복사

아래 내용을 텍스트 편집기에 **그대로 복사**하세요. 빈칸은 본인 연구에 맞게 채웁니다. 완벽하지 않아도 됩니다.

```markdown
# 프로젝트 맥락 문서 (CLAUDE.md)

## 프로젝트
- 주제: ________
- 연구 질문: ________
- 대상/자료: ________
- 방법론: ________
- 현재 단계: [문헌검토 / 자료수집 / 분석 / 작성] 중 ________

## 독자
- 이 연구를 읽을 사람: ________ (학술지 심사위원 / 지도교수 / 정책 관계자)
- 독자가 이 분야에 대해 아는 수준: [전문가 / 비전공자 / 혼합]

## 출력 규칙
- 문체: [학술체 ~이다 / 보고서체 ~합니다 / 구어체]
- 인용 형식: [APA 7th / Chicago / 한국어 학회지 형식]
- 언어: [한국어 / 영어 / 혼용]
- 분량 제한: [문단당 4-6문장 / 전체 3000자 이내 등]

## 금기
- 확인 안 된 논문은 만들지 말 것. 모르면 "확인 필요" 표기.
- 내 연구 질문을 임의로 바꾸지 말 것.
- ________ (본인만의 금기 추가)

## 핵심 선행 연구
- ________ (저자, 연도) — 한 줄 요약
- ________ (저자, 연도) — 한 줄 요약
- ________ (아직 없으면 "탐색 중" OK)
```

### 1-2. **파일로 저장** (여기서 학생이 많이 막히는 지점)

편집기에 내용 작성 후 `CLAUDE.md` 파일로 저장합니다. 위치는 **본인이 찾기 쉬운 폴더** (예: 바탕화면 / 문서 / Downloads / 본인 연구 폴더).

#### Windows — 메모장 사용 시

1. `Ctrl+S` (저장)
2. **파일 형식**: "모든 파일 (*.*)" 선택 ← 중요 (안 바꾸면 `CLAUDE.md.txt`로 저장됨)
3. **파일 이름**: `CLAUDE.md` 입력
4. **인코딩**: "UTF-8" 선택
5. 저장

#### Windows — VS Code 사용 시

1. `Ctrl+S`
2. 파일 이름 `CLAUDE.md` 입력 → 저장. VS Code는 인코딩·확장자를 자동으로 맞춰줍니다.

#### macOS — 텍스트편집기(TextEdit) 사용 시

1. 상단 메뉴 **포맷 → "일반 텍스트로 변환"** 선택 (중요 — 리치 텍스트면 .md로 저장 안 됨)
2. `Cmd+S`
3. 파일 이름: `CLAUDE.md`
4. **"확장자 사용하지 않을 경우"** 질문이 뜨면 "Use .md" 선택
5. 저장

#### macOS / Windows — VS Code 사용 시

1. `Cmd+S` / `Ctrl+S`
2. 파일 이름 `CLAUDE.md` → 저장

### 1-3. 저장 확인

Finder(macOS) 또는 탐색기(Windows)에서 파일을 열어보세요.
- 파일 이름이 정확히 `CLAUDE.md`인지 (`.txt` 같은 꼬리가 붙지 않았는지)
- 내용이 본인이 쓴 마크다운 그대로인지

두 가지 OK면 Step 2로.

> **에디터 없이 하고 싶다면**: Claude 웹 세션 안에 그대로 복붙해서 써도 A/B 비교는 가능합니다. 하지만 파일로 저장해두는 게 **재사용성** 면에서 핵심이에요.

## Step 2 — A/B 비교 실험 

같은 질문을 **맥락 없이 / 맥락 있게** 한 번씩 해서 답변을 비교합니다.

### 2-1. A 세션 (맥락 없음) 

Claude/ChatGPT에 **새 채팅**을 엽니다. 아래만 붙여넣기:

```
내 연구 주제로 서론 도입부를 2문단 써줘.
주제: [본인 주제 한 줄]
```

결과 전체를 별도 창이나 문서에 **복사해서 보관**합니다. (`result_A.txt`로 저장하거나, 메모 앱에 붙여넣기 — 어디든 나중에 비교할 수 있으면 OK)

### 2-2. B 세션 (맥락 있음)

**다른 새 채팅**을 엽니다 (A 세션과 섞이지 않게). 다음처럼 보냅니다:

```
아래는 내 프로젝트 맥락 문서입니다. 모든 답변에서 이 문서를 기준으로 해주세요.

---
[Step 1에서 저장한 CLAUDE.md 전체를 여기에 붙여넣기]
---

위 맥락을 바탕으로 서론 도입부를 2문단 써줘.
```

결과를 `result_B.txt` 또는 메모에 복사 보관.

### 2-3. A vs B 비교 

A와 B를 나란히 놓고 체크하세요:

| 항목 | A (맥락 없음) | B (맥락 있음) |
|---|---|---|
| 문체가 내 연구에 맞나 | | |
| 인용 형식이 맞나 | | |
| 독자 수준에 맞나 | | |
| 다시 수정 요청 없이 쓸 수 있는가 | | |

**차이가 느껴지면 성공.** 맥락 문서가 역할을 하고 있다는 뜻.
**차이가 거의 없으면** — Step 1 문서가 너무 비어 있다는 신호. 돌아가서 구체화하세요 (특히 "출력 규칙", "독자").

> **경험적 패턴**: 무료 챗봇에서도 맥락 문서의 효과는 명확히 보입니다. 단, 맥락 문서가 **구체적**이어야 해요. "학술체" 같은 한 단어보다 "학술체 (~이다). 문단당 4-6문장. APA 7th 인용"처럼 쓰면 차이가 커집니다.

> **📎 본인 결과가 "이게 맞나?" 싶다면**:
> - **완성된 CLAUDE.md 예시** (가상의 석사생 시나리오) → `hands_on/templates/CLAUDE.md.filled_example`
> - **실제 A vs B 응답 비교** (같은 프롬프트, 다른 맥락) → `hands_on/templates/A_vs_B_response_examples.md`
>
> 본인이 받은 응답을 이 예시들의 평가 체크리스트와 대조해 보세요. "어디가 약한가"를 체계적으로 점검할 수 있습니다.

## Step 3 — 이식성 테스트 

> **이 단계가 오늘 실습의 하이라이트입니다.** Step 2가 "맥락 문서가 있으면 좋다"를 보여줬다면, Step 3는 **"왜 채팅창이 아니라 파일로 저장해야 하는가"**(도구를 바꿔도 유지됨)를 손으로 증명합니다.

같은 CLAUDE.md를 **다른 AI 도구**에도 넣어보세요 (Claude → ChatGPT, 또는 반대로).

```
아래 프로젝트 맥락 문서를 읽고, 서론 도입부 2문단을 써줘.
[CLAUDE.md 전체 붙여넣기]
```

- **결과가 비슷하면**: 맥락 문서가 명확하다는 증거. 도구에 의존하지 않는 "자산"이 됐다.
- **결과가 많이 다르면**: 맥락 문서에 모호한 부분이 있다 — 구체적으로 수정.

## 함정 피하기

- **맥락 문서에 모든 걸 넣으려 하지 마세요.** 한 페이지. 자주 반복하는 것만. 부가 정보는 링크로.
- **문서와 실제가 달라지면 바로 고치세요.** 어제의 결정과 오늘의 결정이 다르면, AI는 어제 기준으로 답합니다.
- **"나한테만 통하는 문서"도 가치 있습니다.** 동료에게 넘길 때 다듬으면 됩니다.
- **"학술체로 써줘" 한 줄로 끝내지 마세요.** 예시 한두 개, 금기 두세 개, 인용 형식 명시 — 이 정도는 있어야 AI가 일관되게 적용합니다.

## 올해 달라진 것: Prompt → Context

| 작년 (Prompt 시대) | 올해 (Context 시대) |
|---|---|
| 매번 배경을 복붙 | CLAUDE.md 파일에 한 번 기록 |
| 도구 바꾸면 처음부터 | 같은 문서를 여러 AI에 이식 |
| 결과가 산발적 | 출력 규칙·금기가 일관성 보장 |

**맥락 문서 = harness의 가장 기본 단위.** Track B에서는 같은 원리(맥락을 파일로 남기기)를 **데이터 추출**에 적용합니다.

## (심화 1) Claude Code에서 자동 로드

Claude Code 사용자는 CLAUDE.md가 프로젝트 폴더에 있으면 **자동으로 읽힙니다** — 매번 붙여넣을 필요 없음.

**macOS / Linux**:
```bash
mkdir -p ~/my_research_project
cp CLAUDE.md ~/my_research_project/CLAUDE.md
cd ~/my_research_project
claude
```

**Windows (PowerShell)**:
```powershell
New-Item -ItemType Directory -Path "$HOME\my_research_project" -Force
Copy-Item CLAUDE.md "$HOME\my_research_project\CLAUDE.md"
cd "$HOME\my_research_project"
claude
```

Claude Code 실행 후 대화를 시작하면 CLAUDE.md가 이미 반영된 상태로 응답합니다. 화면 상단에 `CLAUDE.md loaded` 같은 메시지를 확인할 수 있어요.

## (심화 2) AGENTS.md 맛보기

CLAUDE.md가 "이 프로젝트가 뭔가"라면, AGENTS.md는 "어떻게 일할 것인가". 핵심 3줄만:

```markdown
# AGENTS.md (핵심 3줄)
- 애매하면 가정하지 말고 물어라
- 원자료(raw/)는 절대 수정하지 않는다
- 모든 인용에 출처를 붙인다 — 없으면 "확인 필요"
```

전체 템플릿: `hands_on/templates/AGENTS.md.example`. Track C의 에이전트 역할 분담 아이디어와 연결됩니다.
