# Claude Code 연구자용 퀵가이드

> 10분 안에 읽고, 바로 쓸 수 있는 핵심만

---

## 한 줄 요약

**Claude Code = 터미널에서 돌아가는 AI 에이전트.** 파일을 읽고, 수정하고, 명령어를 실행한다. 채팅이 아니라 실행 도구.

---

## 1. 설치 → 시작 (2분)

```bash
npm install -g @anthropic-ai/claude-code   # 설치 (Node.js 18+)
claude auth login                           # 로그인
cd ~/my_research                            # 프로젝트 폴더로 이동
claude                                      # 시작!
```

---

## 2. 반드시 알아야 할 5가지

### ① CLAUDE.md = 프로젝트 설명서

프로젝트 루트에 `CLAUDE.md` 파일을 만들면 **매 세션 시작 시 자동으로 읽는다.** "이 프로젝트가 뭔지" 매번 설명할 필요 없음.

```markdown
# CLAUDE.md 예시

## 프로젝트: 한국 고령화 정책 비교 연구
- 현재 단계: 문헌검토
- 출력 언어: 한국어
- 인용 형식: APA 7th

## 금기
- 확인 안 된 논문 만들지 말 것. 모르면 "확인 필요" 표기
- raw/ 폴더는 절대 수정 금지
```

**만드는 법**: 세션 안에서 `/init` 또는 직접 파일 생성

### ② Plan 모드 — 먼저 계획, 나중에 실행

`Shift+Tab` → plan 모드 전환. Claude가 바로 수정하지 않고 **계획만 먼저** 보여준다.

- 복잡한 작업 전에 반드시 켜기
- 계획 확인 후 `Shift+Tab`으로 다시 실행 모드로

### ③ /compact — 대화가 길어지면 요약

```
/compact                          # 기본 요약
/compact 방법론 관련 내용 위주로   # 초점 지정 요약
```

AI가 갑자기 엉뚱한 소리를 하면 → 컨텍스트가 가득 찼다는 신호. `/compact` 실행.

### ④ /rewind — 마음에 안 들면 되돌리기

```
/rewind    # 이전 시점으로 코드 + 대화 모두 되돌리기
```

"이렇게 해봐" → 결과 확인 → 별로면 `/rewind` → 다른 방향 재시도.

### ⑤ @ 참조 — 파일 직접 지정

```
@data/survey.csv 이 데이터 요약해줘
@paper.pdf 핵심 논점 3개 뽑아줘
```

전체 프로젝트를 스캔하는 대신 **필요한 파일만** 보게 해서 정확도 ↑, 비용 ↓

## 3. 연구자가 자주 쓸 명령어 TOP 10

| 명령어 | 하는 일 |
|---|---|
| `/init` | CLAUDE.md 생성 (프로젝트 초기화) |
| `/plan` | 계획 모드 진입 |
| `/compact` | 대화 요약으로 컨텍스트 확보 |
| `/rewind` | 이전 상태로 되돌리기 |
| `/diff` | 변경사항 확인 |
| `/context` | 컨텍스트 사용량 확인 |
| `/cost` | 토큰 사용량 확인 |
| `/clear` | 대화 초기화 (새로 시작) |
| `/resume` | 이전 세션 이어가기 |
| `/model` | 모델 변경 (sonnet ↔ opus) |

## 4. 실전 예시 3가지

### 예시 A: PDF 논문 10개 일괄 요약

```
sample_data/ 폴더에 PDF 10개가 있어.
각 PDF를 읽고 아래 형식으로 output/summaries/파일명_summary.md 저장해줘:

## 메타
- 제목 / 저자 / 연도 / 학술지
## 핵심 요약 (200자)
## 방법론 한 줄
## 주요 발견 3개
```

### 예시 B: 인터뷰 데이터 코딩

```
transcripts/ 폴더에 인터뷰 텍스트 15개가 있어.
각 파일에서 아래 코딩 축으로 분류해서
output/coding_matrix.md에 표로 정리해줘:

코딩 축: [정책 인식] [서비스 경험] [개선 요구] [정서적 반응]
```

### 예시 C: 파이프로 빠른 분석

```bash
# 한 번 질문하고 결과 받기
claude -p "이 프로젝트에서 .csv 파일 목록과 각 행 수 알려줘"

# 로그 분석
cat experiment_log.txt | claude -p "에러 패턴 요약해줘"

# 변경사항 리뷰
git diff | claude -p "이번 수정 요약해줘"
```

## 5. 핵심 키보드 단축키

| 키 | 기능 |
|---|---|
| `Shift+Tab` | 모드 전환 (기본 → 계획 → 자동) |
| `Escape` | 응답 중단 |
| `Shift+Enter` | 여러 줄 입력 |
| `@` | 파일/URL 참조 |
| `/` | 명령어 목록 |

---

## 6. 이것만 기억하세요

| 상황 | 해결 |
|---|---|
| 매번 같은 걸 설명함 | → CLAUDE.md 쓰세요 |
| AI가 갑자기 이상해짐 | → `/compact` |
| 수정이 마음에 안 듦 | → `/rewind` |
| 큰 작업 시작 전 | → `Shift+Tab`으로 Plan 모드 |
| 어제 하던 작업 계속 | → `claude -c` |
| 특정 파일만 보게 하고 싶음 | → `@파일명` |
